
1- Arquivo: espaço no computador o qual são armazenadas informações, sendo
referido como documentos que eram guardados em ARQUIVOS nas bibliotecas
antigamente, por exemplo.
Eles podem possuir dados estruturados em estrutura de dados ou em registros;
ou podem não ser estruturados, possuindo bytes, o qual indica a quantidade de
memória.
Cabe ressaltar que o formato do arquivo quem decide é quem está executando o
projeto.
2- Arquivo de texto: é um arquivo o qual possui texto simples que são legíveis às
pessoas, servindo para armazenamento de dados no formato JSON; CSV, por
exemplo. O interessante deste arquivo é a possibilidade de poder abrir em
qualquer software, mas deve ter a codificação correta tendo em vista que os
bytes são mapeados através de codificação.
3- Variável: conteúdo o qual varia de valor conforme a alteração no processo da
execução, isto porque ela manipula o código-fonte a fim de executar operações.
Duas das finalidades é evitar que o projetista opere de forma proibida e faz com
que o software trate os dados corretamente.
4- Servidor: computador que possui bancos de memórias, software ou até mesmo
sistema como hard disk internos ou memórias ssd; a fim de rotear serviços que
estão inseridos na rede lan ou wan.

a)	Data center é um local físico onde se encontra computadores e hardeware, assim como servidores, sistema de armazenamento de dados e rede, sendo importante para as empresas por controlar o uso das maquinas de forma mais econômica e ágil. 
Existe data center terceirizado. 
A lei 13709/2018 ( Lei de Proteção de dados) foi criada para proteger os dados pessoais dos cidadãos, assim, os colaboradores devem realizar os projetos conforme a proteção dos dados, utilizando o método Privacy By Design. 
Proteção de danos físicos: investir em segurança local e utilizar ISO 27000 e a PCI DSS., já que os danos físicos podem ter consequências nos dados das pessoas. 
Proteção virtual: utilizar criptografia de dados, firewalls eprogramas antimalware
Para garantir a finalidade da LGPD são destacados os seguintes cuidados: mapear dados, atualizar software e sistema de hardware, verificar recursos,  controlar os acessos.

b) é através dos formatos de arquivos que ocorre a codificação das informações e seu armazenamento e recuperação de arquivo. 

Formatos abertos: Feito por comunidades, sendo utilizado formatos padrões abertos divulgados a todos (pública) e todos podem utilizar sem importar com o criador, assim, os softwares conseguem instaurar em seus sistemas. 
A especificação é publica
Tanto os software livre quanto proprietários devem ter a possibilidade de implantar o formato aberto.
Feito por comunidades

O formato proprietário é utilizada de forma privada, isto é, controlados e desenvolvidas por entidades especificas podendo decidir se tem acesso ou não àquele formato não o contéudo.
Pode ser desenvolvido por empresa ou organização
outros sistemas não teriam capacidade de executar. Uma consequência ruim é caso o software deixe de operar, os dados se perderão
O fato de ser proprietário não quer dizer que gerarão segurança

b.1) formatos proprietários: DWG,SWF, DOC
formatos abertos: MP3, FLAC, OGG

b.2) dados públicos são dados disponibilizados à sociedade por conta de seus interesses, segurança ou prestação de contas; sendo respeitado a boa fe, finalidade e o interesse publico segundo a LGPD.
Pode ser dado coletado ou oriundo do Poder Público.
Pode ser tanto em formato aberto quanto fechado
